## 5.1 什么是“因子”？

在量化投资中，**因子（Factor）本质上就是一个选股的特征或数学公式**。
它接收股票每天的原始数据（比如收盘价、成交量），输出一个**分数**。分数越高，代表这个模型认为这只股票未来越有可能涨。

**一个典型的量价因子公式可能是这样的：**
`Rank(Ts_Mean(Return(Close, 1), 5)) / Rank(Ts_Std(Volume, 20))`
*(用白话说：寻找那些近期收益率排名靠前，但成交量波动却很小、不怎么引人注目的股票。)*

### YAML 配置深度解析

我们可以为这个团队里的每个成员，单独配置不同的大模型。例如，头脑风暴阶段需要发散思维，可以用便宜的模型；写代码和审查阶段需要极高的逻辑性，就用最聪明的模型。

```yaml
# env.yaml
enable_multi_agent: true

llm_agents:
  # 阶段 1：给分析师设置较高的 temperature (0.6)，鼓励他们产生天马行空的想法
  volume_price:
    model: MiniMax-M2.7
    temperature: 0.6
  
  # 阶段 2：首席分析师汇总
  chief_analyst:
    model: kimi-k2.6 
    temperature: 0.6
    timeout_seconds: 120.0
  
  # 阶段 3：生成器和评审员。temperature 调到极低 (0.2)！
  # 写代码不需要创意，需要的是严谨和一丝不苟。
  generator:
    model: kimi-k2.6
    temperature: 0.2
  reviewer:
    model: kimi-k2.6
    temperature: 0.2
```

> **参数说明**：
> - `iteration_count`: 控制让这个团队开几次会。设为 3 表示连着迭代 3 轮。
> - `llm_candidate_count`: 告诉生成器，每轮会议要新产出多少个候选公式。

## 5.3 因子校验：防呆纠错机制 (Step06)

大模型评审员（Reviewer）通过了，就真的能运行了吗？计算机可不吃这一套。
在进入真正的回测之前，Step06 会对这些公式进行无情的**语法与规则审查**：

1. **白名单校验**：大模型是不是生造了一个系统里根本不存在的算子（比如自己发明了一个 `Magic_Function()`）？
2. **字段一致性**：公式里用到了 `pe`，但特征声明里没写？打回重做！
3. **方向判断校验**：系统强制要求大模型在写出公式时，必须标明这个因子是**正向**（分数越高越买）还是**负向**（分数越低越买）。

## 5.4 因子评估与硬筛：选秀的硬指标 (Step07 & Step08)

校验通过的公式，终于可以拿着真实的历史数据计算得分了（Step07）。算完得分后，怎么评判这个因子能不能赚钱？

量化界有几个金标准。我们在 `env.yaml` 中设置了**硬性淘汰门槛**，达不到的因子连上回测跑道的资格都没有：

```yaml
# env.yaml
# 如果因子不满足以下全部条件，直接抛弃跳过回测
min_rank_ic_to_backtest: 0.02      # Rank IC 绝对值阈值
min_rank_ic_ir_to_backtest: 0.2    # Rank IC IR 绝对值阈值
min_positive_ic_ratio: 0.4         # IC 胜率阈值
enable_direction_filter: false     # 是否要求大模型猜的方向必须和历史回测方向一致
```

### 小白科普：这些指标到底是什么鬼？

1. **IC（Information Coefficient，信息系数）**
   - **白话解释**：因子的预测能力。如果 IC 为 1，代表你的分数和未来的涨跌幅 100% 完全对应（你是神仙）；如果 IC 为 0，代表你的分数和扔硬币没区别。
   - **门槛**：`0.02`。别觉得小，在股市里，只要你的预测能力比瞎猜高 2%，并且坚持复利，就能稳定赚钱！
2. **Rank IC（秩信息系数）**
   - **白话解释**：比起预测具体涨了百分之几，量化更关心**排名**。分数排第一的股票，它未来的涨幅是不是也能排第一？这就是 Rank IC。
3. **IR（Information Ratio，信息比率）**
   - **白话解释**：因子的稳定性。如果一个因子今年赚 50%，明年亏 40%，它的 IR 就很低；如果每年稳定赚 10%，它的 IR 就很高。量化机构极度看重高 IR（通常要求 > 0.2）。
4. **胜率（Win Rate / Positive IC Ratio）**
   - **白话解释**：你的因子在过去的 100 次调仓里，有几次是猜对大方向的？系统要求胜率不能偏离 50% 太多（比如设置 `0.4`，低于 40% 说明极度不稳定）。
