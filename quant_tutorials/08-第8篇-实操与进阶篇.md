## 8.1 第一步：Python 环境与依赖包准备

在量化界，Python 是绝对的王者语言。我们需要借助 `conda`（一个强大的 Python 环境管理工具）来创建一个干净的独立空间，防止和其他项目的代码打架。

### 1. 初始化 Conda（仅首次需要）
打开你的 PowerShell（如果你用的是 Windows 10/11），执行以下命令初始化：
*(注意：请将 `D:\miniconda3` 替换为你实际安装 conda 的路径)*
```powershell
& D:\miniconda3\Scripts\conda.exe init powershell
```
**做完这步后，必须关掉当前 PowerShell 窗口，重新打开一个！**

### 2. 创建并激活虚拟环境
```powershell
# 创建一个叫 qlib_v1 的环境，指定 Python 版本为 3.8
conda create -n qlib_v1 python=3.8 -y

# 激活环境（以后每次跑项目前都要执行这句）
conda activate qlib_v1

# 确认环境对不对
python -V
```

### 3. 安装核心依赖包
在激活的 `qlib_v1` 环境中，复制并运行以下命令安装项目所需的“弹药”：
```powershell
# 升级 pip 到最新版
pip install --upgrade pip

# 安装核心依赖
pip install tushare pandas numpy pyarrow openai pydantic PyYAML httpx backtrader
```

如果你想使用酷炫的浏览器图形看板，还需要安装 Streamlit 和画图工具：
```powershell
pip install scipy scikit-learn matplotlib jupyterlab python-dotenv
python -m pip install alphalens-reloaded streamlit -i http://mirrors.aliyun.com/pypi/simple/
```

## 8.3 第三步：一键运行流水线

环境搞定了，钥匙配好了。接下来的操作极其简单，因为系统高度自动化。

请先确保你的终端路径在项目源码目录下：

### 动作 1：寻找最佳训练窗口（可选前置）

## 8.4 厂长避坑指南（常见问题排查）

1. **Tushare 接口频频报错 / 限流怎么办？**
   - 现象：跑到一半提示“抱歉，您每分钟最多访问XX次”。
   - 解决：这是因为你把“股票池”设得太大（比如 `all_market` 全市场 5000 只股）。**新手调试强烈建议在 `env.yaml` 中把股票池设为 `random_sample`（随机抽 100 只）**。等跑通全流程后，睡前再开大股票池让电脑自己跑通宵。
2. **大模型一直不输出公式，一直在重试？**
   - 现象：日志里全是 timeout 或者 parse error。
   - 解决：可能是你填写的 API Key 欠费了，或者网络连不上 `base_url`。也可能是大模型“不够聪明”，无法按照要求的 JSON 格式输出。建议把 `generator` 的模型换成更聪明的版本。
3. **回测没反应，提示被跳过了？**
   - 现象：Step08 提示 `skipped`。
   - 解决：大概率是因子的 IC 和胜率太烂，被 `env.yaml` 里的硬筛条件（如 `min_rank_ic_to_backtest`）给无情淘汰了。这是正常现象，说明大模型这把没发挥好，增加 `iteration_count` 让它多试几次！
