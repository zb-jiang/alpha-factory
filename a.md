### 一、 阶段一：核心市场状态感知与 Prompt 升级（当前目标）

**1. 引入“训练环境 (Train)”与“测试环境 (Test)”的双重上下文**
与其给一个笼统的训练集总结，不如将市场状态切分为“实验室环境”与“实战目标环境”告诉 LLM。因为线下选定的 Train 区间代表了挖掘规律的土壤，而 Test 区间代表了因子即将面对的真实战场。
- **训练环境上下文（Train Context）**：基于 `train_start_date` 到 `train_end_date` 的数据。告诉 LLM 特征数据是在什么样的市场土壤里长出来的（例如：“该阶段大部分时间处于缩量震荡市”）。这能防止 LLM 纯粹迎合训练集数据的表面相关性而陷入过拟合。
- **测试环境上下文（Test Context）**：基于 `test_start_date` 之前的近期市场切片。明确因子将要部署的战场环境（例如：“当前目标环境流动性大幅萎缩，高换手率的小盘股领涨”）。提示 LLM 对比两者的异同，在环境相似时大胆使用趋势动量逻辑，在环境突变时加入防御性逻辑（如控制高波、换手惩罚等），从而实现对当前投资有指导意义的“定向爆破”。

**1.1 如何科学地选取 Train（训练环境）的时间段？（非常重要）**
你提到“如何通过线下方式基于当前的市场环境选取能够指导当前投资的训练上下文”，这是一个极其专业的量化实战问题。盲目地取过去三年作为训练集往往效果很差，因为市场风格在不断切换。以下是业界主流的**三种选段策略**：

- **策略 A：寻找“历史镜像区间”（相似性选段，最推荐）**
  1. 先算出现阶段（Test 之前 1 个月）的**市场状态向量**（比如：`[低波动, 缩量, 小盘占优, 高离散度]`）。
  2. 写一个小脚本，遍历过去 10 年的数据，计算历史上每个月的状态向量。
  3. 计算历史各月与当前状态的**欧氏距离**或**余弦相似度**。
  4. 挑出历史上与当前最相似的 3-4 个时间段（比如：2018年下半年、2021年二季度），把它们拼接起来，或者直接选包含这些相似月份最多的一整年作为 Train 区间。
  **逻辑**：在相似的宏观环境下，Alpha 逻辑大概率会重演。

- **策略 B：“压力测试区间”（互斥选段）**
  如果你想让 LLM 挖掘**高泛化性、高防御性**的因子，你反而不能选相似区间。
  1. 看看现在的 Test 是什么状态（比如：当前是“低波缓慢上涨”）。
  2. 故意去历史数据中挑一段**完全相反**的恶劣环境作为 Train（比如：挑一段“高波暴跌”的股灾区间）。
  **逻辑**：如果 LLM 在这种地狱模式下都能挖出不亏钱的因子，那这个因子拿到现在的温和环境里，鲁棒性极强。这叫“把兵练在最苦的地方”。

- **策略 C：“近期动量延续区间”（就近选段）**
  这是最简单但也最容易失效的方法。
  1. 假设现在是 2026 年 4 月，你认为当前的这种“小盘微盘炒作风气”是从 2025 年 10 月才开始的，并且还在延续。
  2. 那你就直接把 Train 区间定为 2025-10-01 到 2026-03-31。
  **逻辑**：假设短期内市场风格不会发生根本性逆转（风格动量）。缺点是样本数据太少，容易导致 LLM 过拟合少数几只妖股的特征。

**建议落地方式**：你可以先在本地（不调用 LLM）利用我们即将写的合成指数算一下历史 5 年每天的 `[趋势, 波动, 成交, 风格, 离散度]`，然后做个简单的 KNN（K近邻）或者直接肉眼看数据，找到和当前最新一周状态最像的历史片段，把这个片段的起止时间填进 `analysis_rule.yaml` 里。这就是用**数据驱动代替拍脑袋**来选 Train 区间。

**2. 增加“横截面离散度（Cross-sectional Dispersion）”维度**
在量价因子挖掘中，这是一个极其重要的宏观变量，但很多人会忽略：
- 它是指**每天股票池内所有股票收益率的标准差**。
- **高离散度**：个股走势严重分化，Alpha 极好做，选股因子（如动量、反转）表现极佳。
- **低离散度**：所有股票同涨同跌（比如暴跌时的千股跌停，或暴涨时的普涨），此时选股因子基本失效，Beta 占主导。
- **如何应用**：**两段数据都需要计算并做对比。**
  - **Train 中的离散度**：告诉 LLM 模型学习的这段历史数据中，Alpha 的“丰度”是如何的。如果 Train 阶段离散度一直很低，LLM 挖掘出来的可能是偏向 Beta 择时的因子。
  - **Test（实战）中的近期离散度**：告诉 LLM 即将面对的真实战场，Alpha 是否好做。如果 Test 近期离散度极高，提示 LLM 重点挖掘横截面选股 Alpha（如反转、动量）；如果 Test 近期离散度极低，提示 LLM 挖掘能抵御系统性风险的防御因子或大盘风格因子。
  - 将两者对比，LLM 就能知道：“哦，之前训练的这段时间个股分化很大，所以选股因子管用，但现在实战环境大家同涨同跌，我得改变策略。”

**3. 利用“自下而上”的方法合成市场状态（免去引入外部指数）**
你的股票池配置是 `SH000300`（沪深300）。为了最真实地反映**你所交易的这个池子**的市场状态，你不必去额外下载外部的大盘指数数据。
- 既然系统中已经加载了这 300 只股票的日频面板数据（`raw_frame`），你完全可以每天算一个**股票池等权收益率**或**市值加权收益率**。
- 用这个“合成指数”来计算趋势、波动和回撤，它比外部宽基指数更贴切，因为它完全反映了你**当下能买到的股票**的真实生态。
- **这正是你计划书中 5 个维度及建议输出形式的完美落地方式（映射关系表）**：
  
  | 计划书中的维度 | 用合成指数方法的落地方式 | 建议输出形式（给LLM的离散化文本） |
  | :--- | :--- | :--- |
  | **趋势状态** | **`market_return` 的时序累加**。合成指数的 `rolling(20).sum()` 就是 20 日收益率；`rolling(60).mean()` 就是均线；用最高点比当前点可算回撤。 | 当前区间整体偏**上行 / 下行 / 震荡** |
  | **波动状态** | **`market_return` 和 `cross_dispersion`**。合成指数的 `rolling(20).std()` 就是指数波动率；每天的 `cross_dispersion` 就是横截面波动率（即涨跌分化度）。 | 当前阶段波动处于**高 / 中 / 低**水平 |
  | **成交状态** | **`market_turnover` 和 `market_amount`**。把这两者 `rolling(250).rank(pct=True)` 就能直接得出当前换手和成交额在过去一年的历史分位数。 | 当前阶段成交处于**活跃 / 一般 / 冷清** |
  | **风格状态** | **分组聚合（Groupby）计算收益差**。在截面聚合时，按市值大小分两组算收益率，两者相减即为大盘相对小盘的风格收益（同理可做高低波）。 | 市场风格偏**大盘 / 小盘，偏趋势 / 反转** |
  | **广度状态** | **在截面聚合时直接按条件统计**。比如在 `groupby('date')` 时，统计 `(ret_1d > 0).mean()`，这就是当天的上涨家数占比。 | （用于辅助判断趋势真伪与行情广度） |
  | **历史相似性** | **状态向量的距离计算**。将上述维度当前值组合成向量，与历史截面的状态向量计算欧氏距离或相关性。 | 与**历史上若干区间（如XXXX年X月）**存在相似性 |

**4. 强制 LLM 进行“失效场景预判”（Expected Failure Regime）**
在 LLM 输出的 JSON 格式中，除了现有的 `reason`（逻辑解释）和 `risk`（统计风险），强烈建议增加一个独立字段：`expected_failure_regime`（预期失效环境）。

**什么是失效场景预判？为什么重要？**
没有任何一个量化因子是全天候赚钱的。因子赚钱的本质是承担了某种特定的市场风险或利用了某种特定的市场错误。因此，**知道一个因子什么时候会亏钱，比知道它为什么能赚钱更重要**。

- **倒逼 LLM 深入思考逻辑**：
  如果 LLM 只是随机拼凑了一个 `(close - low) / volume` 的公式，它是写不出明确的失效场景的。但如果它是基于“均值回复”逻辑写的公式，它就能明确写出：“当市场出现单边趋势暴涨或连续逼空时，该因子会持续做空强势股，从而产生巨大回撤。”
- **与 Market Context 形成闭环**：
  我们费这么大劲把 Train 和 Test 的市场环境（高低波、趋势、离散度）算出来喂给 LLM。让 LLM 预判因子的失效环境，就是检查它有没有把这套市场语言学进去。
- **为“因子组合分配（Portfolio Allocation）”提供元数据**：
  假设 LLM 挖出了 10 个 IC 都不错的因子，并保存了它们各自的 `expected_failure_regime`。
  当未来某一天，我们的系统监测到“市场波动率突然放大”时，我们就可以直接写个脚本，把那些被 LLM 标记为“在高波环境下容易失效”的因子权重调低，甚至一键关停。这为未来实现动态的因子权重分配打下了极其重要的数据基础。

**5. 阶段一落地架构建议**
为了不污染现有的 `step03_health_check.py`（它目前主要聚焦于特征本身的统计和共线性），我建议：
1. **计算层**：在 `step04_build_summary.py` 中增加一段逻辑，或者干脆加一个 `step03b_market_context.py`。输入是现成的 `raw_frame`，基于 `date` 进行 `groupby`，算出上述的趋势、波动、离散度、换手率分位数等时序数据。
2. **规则引擎层**：写几个简单的阈值规则（比如：最近20日日均成交额 > 过去250日均值的 1.5 倍 -> 映射为文本标签 `[极度活跃]`）。
3. **输出层**：生成 `market_context.json`，最终在 `step05_call_llm.py` 里无缝拼接到 `user_prompt` 中。
- **具体计算步骤**：
  1. **第一步：截面聚合（Cross-sectional Aggregation）得到“每日市场画像”**
     在提取特征数据或构建 Summary 的阶段（如新增的 `step04b_market_context.py` 中），直接读取清洗好的面板数据 `raw_frame`。
     执行一次按日期（`date`）的 `groupby`：
     ```python
     # 假设 raw_frame 包含 'ret_1d', 'turnover', 'amount'
     daily_market = raw_frame.groupby('date').agg(
         market_return=('ret_1d', 'mean'),        # 合成指数单日收益
         cross_dispersion=('ret_1d', 'std'),      # 横截面离散度
         market_turnover=('turnover', 'mean'),    # 市场平均换手率
         market_amount=('amount', 'sum')          # 市场总成交额
     )
     ```
     这样就得到一个纯时序的 DataFrame（Index 是日期），相当于自己用股票池“捏”出了一个包含深度信息的合成指数。

  2. **第二步：时序滚动（Time-series Rolling）计算市场状态变量**
     在时间轴上做滚动计算，得出需要的宏观状态：
     ```python
     # 计算市场趋势（20日累计收益）
     daily_market['trend_20d'] = daily_market['market_return'].rolling(20).sum()
     # 计算市场波动率（20日收益率标准差）
     daily_market['volatility_20d'] = daily_market['market_return'].rolling(20).std()
     # 计算当前成交额相对过去一年的分位数（活跃度）
     daily_market['amount_rank_250d'] = daily_market['market_amount'].rolling(250).apply(
         lambda x: pd.Series(x).rank(pct=True).iloc[-1]
     )
     ```

  3. **第三步：时间切片输出**
     最后，根据 `train_end_date` 和 `test_start_date`，在这个算好的 `daily_market` 中切出两段时间：
     - **对 Train 阶段**：把这段时间里的 `trend_20d`、`volatility_20d` 等指标求平均或做分布统计，生成描述实验室土壤的文字总结。
     - **对 Test 阶段**：直接取 `test_start_date` 前最后一天（或最后一周）的那一行数据，翻译成当前最新的实战状态描述。
     输出一个 `market_context.json`，供下一步 LLM 读取，彻底摆脱对外部宏观指数的依赖。

### 二、 阶段二：因子自适应进化（进阶目标）

**1. 引导 LLM 生成“条件触发型”因子（Regime-Conditional Factors）**

**什么是条件触发型因子？**
传统的量化因子是“全天候”开启的，比如因子公式 `ret_20d`（20日收益率），它在牛市和熊市里都在无差别地买入强势股。
而“条件触发型因子”就像自带了一个“智能开关”或“动态杠杆”，它能**把宏观市场状态直接写进因子公式里**。

- **举个例子：动态防守因子**
  假设我们把刚才算出来的“市场波动率（`market_vol_20d`）”作为基础特征也提供给 LLM。
  LLM 就可以写出这样一个因子公式：
  `ret_20d / market_vol_20d`
  **它的逻辑是**：在市场风平浪静（`market_vol_20d` 极低）时，公式分母很小，因子暴露度极高，系统会满仓去追逐动量（`ret_20d`）；一旦市场发生暴跌、波动率飙升（`market_vol_20d` 极大），分母变大，整个因子的绝对值会被极度压缩，从而**自动降低仓位暴露**，实现防守。

- **如何实现？**
  1. 在我们用“合成指数”算出 `market_return`, `market_vol_20d`, `cross_dispersion` 等宏观状态序列后，我们不仅要用它们来生成“文字总结”，还可以把它们 `merge`（拼接）到每天的个股截面数据中（让每只股票在同一天都拥有相同的宏观状态值）。
  2. 在 `feature_pool.yaml` 中把这些宏观变量注册为合法的 `base_features`。
  3. 在 LLM 的提示词中鼓励它：“既然你已经拥有了 `market_vol_20d` 这样的市场状态特征，请尝试将它作为条件项或惩罚项写入你的因子公式中，让因子具备环境自适应能力！”
