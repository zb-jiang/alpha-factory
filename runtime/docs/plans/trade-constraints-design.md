# 交易约束规则设计（全策略通用）

> 本文档定义了所有策略（TopKDropout、EnhancedIndexing 等）在交易执行层面的统一约束规则。各策略设计文档中不再单独描述停牌/涨停/跌停处理，统一引用本文档。

## 核心原则

| 约束类型 | 调仓日行为 | 后续交易日行为 | 延续范围 |
|---------|-----------|--------------|---------|
| **停牌** | 跳过，不做交易 | 不重试 | 无延续 |
| **涨停** | 跳过，不做交易 | 不重试 | 无延续 |
| **跌停** | 跳过，不做交易 | **每日重试卖出** | 延续到下一个调仓日 |

## 停牌规则

**含义**：股票当日无法交易（无对手方）。

**操作**：
- 调仓日：跳过该股票，不做任何交易
- 后续交易日：不重试，等待下一个调仓日重新决策

**设计理由**：
- 停牌是外部强制的交易禁止，无法主动解决
- 停牌期间无法判断何时恢复，重试无意义
- 下一个调仓日会重新计算，自然处理恢复交易的股票

---

## 涨停规则

**含义**：股票当日涨停，买入挂单可能无法成交。

**操作**：
- 调仓日：跳过该股票的买入，不做交易
- 后续交易日：不重试，等待下一个调仓日重新决策

**设计理由**：
- 涨停不买是"少赚"而非"多亏"，风险可控
- 涨停打开后价格可能已大幅偏离目标价位，此时买入可能不划算
- 不延续到后续交易日，避免在非调仓日产生意外交易

---

## 跌停规则

**含义**：股票当日跌停，卖出挂单可能无法成交。

**操作**：
- 调仓日：跳过该股票的卖出，不做交易
- 后续交易日：**每日重试卖出**，直到卖出成功或到达下一个调仓日
- 下一个调仓日：独立重新决策，不再延续之前的卖出意图

**设计理由**：
- 跌停卖不出是"多亏"而非"少赚"，风险不对称
- 跌停打开后应尽快卖出止损，每日重试是合理的风控行为
- 真实交易中，交易员一定会次日继续尝试卖出
- 延续范围有限：仅到下一个调仓日，不无限延续

### 跌停场景示例

```
调仓日 A（周一）：策略决定卖出股票 X，但 X 跌停无法卖出
  周二：重试卖出 X → 仍跌停，失败
  周三：重试卖出 X → 打开跌停，卖出成功 ✓

调仓日 B（下周一）：独立重新计算，不考虑 A 日的卖出意图
```

```
调仓日 A（周一）：策略决定卖出股票 X，但 X 跌停无法卖出
  周二~周五：每天重试，均跌停无法卖出

调仓日 B（下周一）：独立重新计算
  - 若 X 仍在卖出名单中 → 继续尝试卖出
  - 若 X 不在卖出名单中（如排名回升）→ 不卖出，保留持仓
```

## 三种约束的风险对比

| 维度 | 停牌 | 涨停 | 跌停 |
|------|------|------|------|
| 想做但做不了 | 买或卖 | 买入 | 卖出 |
| 不操作的后果 | 无法交易 | 少赚（机会成本） | 多亏（实际风险） |
| 风险性质 | 中性 | 低风险 | 高风险 |
| 是否延续重试 | 否 | 否 | 是 |
| 延续范围 | - | - | 到下一个调仓日 |

---

## 与调仓日独立性原则的关系

> **调仓日独立性原则**：每个调仓日独立决策，不关联前序调仓日的意图。

跌停的 `delay_sell` 延续重试**不违背独立性原则**：

- **独立性**是对**决策层面**而言的：每个调仓日独立决定"卖什么、买什么"
- **`delay_sell`** 是**执行层面**的延续：决策已经做出（卖出），只是因为市场原因无法执行，所以延续执行
- **边界明确**：延续仅到下一个调仓日，到达后重新独立决策

---

## 配置参数

```yaml
Execution:
  suspend_action: skip          # 停牌处理：skip（跳过，不延续）
  limit_up_action: skip_buy     # 涨停处理：skip_buy（跳过买入，不延续）
  limit_down_action: delay_sell # 跌停处理：delay_sell（跳过卖出，后续每日重试直到下一个调仓日）
```
