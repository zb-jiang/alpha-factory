# SoftTopK 策略业务场景设计

## 当前配置参数

```yaml
SoftTopK:
  holding_count: 30       # 持仓股票数（按因子分从高到低选前N只）
  weight_func: softmax    # 权重函数：softmax 或 rank_power
  softmax_temperature: 1.0  # softmax 温度参数（T 越小集中度越高）
  rank_power_alpha: 1.0    # rank_power 衰减参数（α 越大头部越集中）
  min_score_coverage: 0.90  # 因子分数保有率阈值（低于此值本调仓日不操作）
```

## 策略概述

**SoftTopK（软性TopK策略）** 的核心逻辑：

1. **选股**：从股票池中，按因子得分从高到低排序，选前 `holding_count` 只
2. **权重分配**：根据因子得分连续分配权重（而非简单等权），使"高置信度信号"获得更高仓位
3. **无缓冲机制**：不考虑当前持仓，每个调仓日全量重算

## 核心参数说明

### `holding_count` - 持仓股票数

**含义**：按因子得分从高到低选前 N 只股票持仓。

**影响**：
- N 越大（如 50~80）：越分散，头部集中度越低
- N 越小（如 15~20）：越集中，头部股票权重更大

### `weight_func` - 权重函数

支持两种权重分配方式：

#### `softmax`（默认）

- **公式**：`weight_i = exp(score_i / T) / Σ exp(score_j / T)`
- **特点**：对高分更敏感，头部集中度更高
- **温度参数** `softmax_temperature`（默认 1.0）：
  - T 越小：集中度越高（接近赢家通吃）
  - T 越大：越接近等权
- **直觉**：得分差异越大，权重差异越大

#### `rank_power`

- **公式**：`weight_i = 1 / rank_i^α`
- **特点**：按排名幂律衰减，权重更平滑、稳健
- **衰减参数** `rank_power_alpha`（默认 1.0）：
  - α = 1：标准调和衰减
  - α > 1：衰减更快，头部更集中
  - α < 1：衰减更慢，更接近等权
- **直觉**：不依赖原始分数值，只依赖排名，对异常值更鲁棒

### `min_score_coverage` - 因子分数保有率阈值

**含义**：调仓日因子分数非缺失的股票数占股票池总数的最低比例。

**作用**：
- 若保有率 < `min_score_coverage`，本调仓日不做任何操作
- 若保有率 ≥ `min_score_coverage`，缺失因子的股票不参与排名

## 场景列举

### 选股规则说明

> **核心规则**：每个调仓日，从股票池中按因子得分从高到低排序，取前 `holding_count` 只股票作为目标持仓。**无缓冲区，不考虑当前持仓**。

### 场景 1：初始建仓

**条件**：当前持仓为 0 只股票

**操作**：
- 按因子得分排名选前 `holding_count` 只股票
- 按 `weight_func` 计算权重
- 按照目标权重建仓

**权重计算示例（softmax，holding_count=5，temperature=1.0）**：
```
因子得分：[1.2, 0.9, 0.6, 0.3, 0.1]
softmax 权重：[31.4%, 23.3%, 17.3%, 12.8%, 10.5%]
→ 高分股票（1.2）权重约为低分股票（0.1）的 3 倍
```

**权重计算示例（rank_power，holding_count=5，alpha=1.0）**：
```
排名：[1, 2, 3, 4, 5]
rank_power 权重：[43.8%, 21.9%, 14.6%, 10.9%, 8.8%]
→ 第1名权重约为第5名的 5 倍
```

---

### 场景 2：因子得分全部相同

**条件**：
- 前 N 只股票因子得分完全相同

**操作**：
- softmax：所有股票权重相等（exp 相同 → 均分）
- rank_power：按排名衰减（不依赖原始分数值）
- 退化结果：接近等权

---

### 场景 4：因子分数保有率不足

**条件**：
- 调仓日因子分数非缺失的股票数 < `min_score_coverage` × 股票池总数

**操作**：
- 不做任何交易，保持当前持仓不变
- 因子数据质量不足，排名不可靠

---

### 场景 5：因子分数保有率充足但部分缺失

**条件**：
- 调仓日因子分数保有率 ≥ `min_score_coverage`
- 部分股票因子分数缺失

**操作**：
- 缺失因子的股票不参与排名
- 用剩余有效股票的因子分数进行排名和权重计算

---

### 场景 6：softmax 温度参数的影响

**条件**：
- 同一组因子得分，不同的 `softmax_temperature`

**效果**：
```
得分：[1.0, 0.8, 0.6, 0.4, 0.2]

T = 0.5（低温）：[42.9%, 26.0%, 15.8%, 9.5%, 5.8%]  → 高度集中
T = 1.0（默认）：[27.6%, 22.6%, 18.5%, 15.2%, 11.8%] → 适度集中
T = 2.0（高温）：[18.4%, 17.0%, 15.7%, 14.5%, 13.4%] → 接近等权
```

---

### 场景 7：rank_power 衰减参数的影响

**条件**：
- 同一组排名，不同的 `rank_power_alpha`

**效果**：
```
排名：[1, 2, 3, 4, 5]

α = 0.5（慢衰减）：[27.2%, 19.2%, 15.7%, 13.6%, 12.1%] → 接近等权
α = 1.0（默认）：[43.8%, 21.9%, 14.6%, 10.9%, 8.8%]  → 标准衰减
α = 2.0（快衰减）：[64.3%, 16.1%, 7.1%, 4.0%, 2.6%]  → 高度集中
```

## 核心业务逻辑澄清

### 权重计算流程

1. **数据质量检查**：因子分数保有率 < `min_score_coverage` → 不操作
2. **股票选择**：按因子得分排名取前 `holding_count` 只
3. **权重计算**：按 `weight_func`（softmax 或 rank_power）计算连续权重

### 两种权重函数对比

| 维度 | softmax | rank_power |
|------|---------|------------|
| 依赖原始分数值 | 是 | 否（仅依赖排名） |
| 对异常值敏感度 | 较高 | 较低 |
| 集中度可调性 | 通过温度参数 | 通过衰减参数 |
| 适用场景 | 因子得分分布均匀、差异可信 | 因子得分波动大、排名更可靠 |

## 已确认的业务逻辑

1. **因子分数保有率检查**：保有率 < `min_score_coverage` 时本调仓日不操作
2. **选股规则**：按因子得分排名前 `holding_count`，无缓冲区
3. **权重分配**：连续分配（非等权），高分股票权重更大
4. **因子数据缺失处理**：因子得分缺失的股票不参与排名
5. **交易约束**：停牌跳过不延续、涨停跳过不延续、跌停跳过但后续每日重试（详见 [trade-constraints-design.md](trade-constraints-design.md)）
6. **调仓日独立性**：每个调仓日独立计算

## 最终业务逻辑总结

1. **因子分数保有率**：保有率 < `min_score_coverage` 时本调仓日不操作
2. **股票选择**：按因子得分排名前 `holding_count` 的股票（无缓冲机制）
3. **权重分配**：softmax（按分数指数分配）或 rank_power（按排名幂律衰减）
4. **因子缺失**：因子得分缺失的股票不参与排名
5. **交易约束**：停牌/涨停跳过不延续，跌停每日重试直到下一个调仓日（详见 [trade-constraints-design.md](trade-constraints-design.md)）
6. **调仓日独立性**：每个调仓日独立计算
