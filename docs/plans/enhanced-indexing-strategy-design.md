# EnhancedIndexing 策略业务场景设计

## 当前配置参数

```yaml
EnhancedIndexing:
  active_weight_bound: 0.02        # 单只股票主动权重偏离上限（相对基准权重 ±2%）
  holding_count: 30               # 组合目标持仓数（从指数成分股中选前N只）
  weight_mode: benchmark_tilt      # 权重模式：z-score 倾斜（唯一模式）
  min_score_coverage: 0.90         # 因子分数保有率阈值（低于此值本调仓日不操作）
  min_weight_coverage: 0.90        # 成分权重保有率阈值（低于此值本调仓日不操作）
```

## 策略概述

**EnhancedIndexing（增强型指数策略）** 的核心逻辑：

1. **选股**：从指数成分股中，按因子得分从高到低排序，选前 `holding_count` 只
2. **基准权重**：以指数真实成分股权重作为每只股票的基准权重
3. **权重倾斜**：在基准权重基础上，用因子得分的 z-score 做有约束的倾斜
4. **风险控制**：单只股票的主动权重偏离不超过 `active_weight_bound`

## 核心参数说明

### `holding_count` - 目标持仓数

**含义**：从指数成分股中选出因子得分排名前 N 的股票持仓。

**示例**：指数 = 沪深300（300只），holding\_count = 30 → 选因子得分前 30 名

**影响**：

- 数量更大（如 50）：覆盖更多成分股，跟踪更紧密，换手更平滑
- 数量更小（如 20）：Alpha 更集中，但跟踪误差可能增大

### `active_weight_bound` - 主动权重偏离上限

**含义**：单只股票相对其基准权重的最大增减幅度（如 ±2%）。

**示例**：某股票基准权重 5%，`active_weight_bound` = 0.02 → 实际权重范围 \[3%, 7%]

**作用**：

- 防止单票权重过度倾斜导致集中度失控（风控作用）
- 注意：由于归一化的存在（见场景3），`active_weight_bound` 控制跟踪误差的效果会被稀释，但它作为单票倾斜上限的风控作用仍然有效
- `holding_count` 对跟踪误差的影响远大于 `active_weight_bound`

**`holding_count`** **与** **`active_weight_bound`** **的关系**：

- `holding_count` 决定了选股范围，是Alpha的主要来源，也决定了归一化放大倍数（跟踪误差的主因）
- `active_weight_bound` 决定了在基准权重基础上做微调的幅度，是额外增强，也是单票集中度的安全阀
- 两者不矛盾：`holding_count` 管大方向，`active_weight_bound` 管微调和风控

### `weight_mode` - 权重模式

当前仅支持一种模式：**`benchmark_tilt`**

- **基准权重**：指数真实成分股权重（如沪深300中茅台占5%，某小票占0.1%）
- **倾斜方式**：对入选股票的因子得分做 z-score 标准化，乘以 `active_weight_bound`
- **公式**（分4步）：
  1. 基准权重 = 股票在指数中的真实权重（如茅台在沪深300中占5%）
  2. z-score = (该股票因子得分 - 入选股票平均分) / 入选股票标准差
  3. 倾斜量 = z-score × active\_weight\_bound，然后截断到 \[-active\_weight\_bound, +active\_weight\_bound]
  4. 最终权重 = 基准权重 + 截断后的倾斜量
- **直觉**：因子得分高的股票在基准权重上多配一点（最多+2%），因子得分低的少配一点（最多-2%），既利用信号又控制偏离
- 非入选股票权重归零，其权重重新分配给入选股票

### `min_score_coverage` - 因子分数保有率阈值

**含义**：调仓日因子分数非缺失的股票数占指数成分股总数的最低比例。

**计算**：`保有率 = 因子分数非缺失的成分股数 / 指数成分股总数`

**作用**：
- 若保有率 < `min_score_coverage`，本调仓日不做任何操作（排名都做不了，后续动作没有意义）
- 若保有率 ≥ `min_score_coverage`，缺失因子的股票不参与排名

**示例**：沪深300有300只成分股，`min_score_coverage` = 0.90
- 270只以上有因子分数 → 正常操作
- 仅260只有因子分数（260/300=86.7% < 90%）→ 本调仓日不操作

### `min_weight_coverage` - 成分权重保有率阈值

**含义**：入选的前 `holding_count` 只股票中，拥有指数成分权重数据的股票数占入选总数的最低比例。

**计算**：`保有率 = 入选股票中有成分权重数据的数量 / holding_count`

**作用**：
- 若保有率 < `min_weight_coverage`，本调仓日不做任何操作（权重数据都没有，做动作没有意义）
- 若保有率 ≥ `min_weight_coverage`，缺失权重的股票使用等权 `1/n` 作为 fallback

**示例**：holding_count = 30，`min_weight_coverage` = 0.90
- 27只以上有权重数据 → 正常操作，无权重数据的股票用 1/30 作为 fallback
- 仅26只有权重数据（26/30=86.7% < 90%）→ 本调仓日不操作

## 场景列举

### 选股规则说明

> **核心规则**：每个调仓日，从指数成分股中按因子得分从高到低排序，取前 `holding_count` 只股票作为目标持仓。**无缓冲区，不考虑当前持仓**。
>
> 这意味着：如果某只股票从第 31 名升到第 30 名，它就会进入持仓；如果从第 30 名跌到第 31 名，就会立刻被移出。**没有类似 TopKDropout 的** **`sell_drop_to`** **缓冲机制来降低换手率**。

### 场景 1：初始建仓

**条件**：当前持仓为 0 只股票（指数 = 沪深300，holding\_count = 30）

**操作**：

- 对沪深 300 全部成分股按因子得分从高到低排序
- 选出排名前 30 的股票
- 读取这 30 只股票在指数中的真实成分股权重作为基准权重
- 用 z-score 倾斜计算目标权重
- 按照目标权重建仓

**权重计算示例**：

```
假设入选的3只股票（简化示例）：
  茅台：基准权重 5.0%，z-score = 1.5
  平安：基准权重 1.2%，z-score = -0.5
  招行：基准权重 0.8%，z-score = 0.3

active_weight_bound = 0.02

  茅台：tilt = 1.5 × 0.02 = 0.03 → clip后 = 0.02
        最终权重 = 5.0% + 2.0% = 7.0%

  平安：tilt = -0.5 × 0.02 = -0.01 → clip后 = -0.01
        最终权重 = 1.2% - 1.0% = 0.2%

  招行：tilt = 0.3 × 0.02 = 0.006 → clip后 = 0.006
        最终权重 = 0.8% + 0.6% = 1.4%
```

***

### 场景 2：排名微变导致换手

**条件**：

- 当前持仓 30 只股票（因子排名前 30）
- 新调仓日，第 30 名股票 A 跌到第 31 名，第 31 名股票 B 升到第 30 名

**操作**：

- 卖出股票 A（跌出前 30 名，无缓冲保护）
- 买入股票 B（进入前 30 名），使用股票 B 在指数中的真实权重作为基准
- **即使排名仅差 1 位，也会触发换手**（与 TopKDropout 不同）

***

### 场景 3：权重归一化处理

**条件**：

- 入选 30 只股票的基准权重之和 < 100%（因为未入选的 270 只股票权重被释放）

**操作**：

- 非入选股票权重归零
- 入选股票的倾斜后权重需要归一化，使总权重之和为 100%
- 归一化方式：等比例缩放所有入选股票的最终权重

**归一化的影响**（以沪深300，holding\_count=30为例）：

```
入选30只股票的基准权重之和 ≈ 55%（其余270只占45%）
归一化前：茅台基准权重 5%，权重之和 ≈ 56%
归一化后：5% / 56% ≈ 8.9%，所有入选股票权重等比例放大

关键认识：归一化本身就会放大跟踪误差。
holding_count越小，入选股票基准权重之和越小，归一化放大倍数越大，跟踪误差越大。
如果希望严格控制跟踪误差，应设置较大的 holding_count（如50~100）。
```

***

### 场景 4：单只股票因子得分极端

**条件**：

- 股票 A 的 z-score = 3.0（极端高分）
- 基准权重 4.0%，active\_weight\_bound = 0.02

**操作**：

- tilt = 3.0 × 0.02 = 0.06 → clip 后 = 0.02
- 最终权重 = 4.0% + 2.0% = 6.0%
- **clip 确保偏离不超过 ±2%，无论因子得分多极端**

***

### 场景 5：因子得分分布过于集中

**条件**：

- 入选 30 只股票的因子得分标准差非常小（std < 1e-12）
- 所有股票得分相近，无明显差异

**操作**：

- 采用纯基准权重配置，不做倾斜
- 本质上退化为指数抽样复制

***

### 场景 6：因子分数保有率不足

**条件**：
- 调仓日因子分数非缺失的成分股数 < `min_score_coverage` × 成分股总数
- 例如：沪深300仅260只有因子分数（260/300=86.7% < 90%）

**操作**：
- 不做任何交易，保持当前持仓不变
- 因子数据质量不足，排名不可靠，后续操作没有意义

---

### 场景 7：因子分数保有率充足但部分缺失

**条件**：
- 调仓日因子分数保有率 ≥ `min_score_coverage`（如 280/300=93.3%）
- 20只成分股因子分数缺失

**操作**：
- 缺失因子的股票不参与排名
- 用剩余280只股票的因子分数进行排名，选前 `holding_count` 只

---

### 场景 8：成分权重保有率不足

**条件**：
- 入选的30只股票中，仅26只在指数成分权重数据中找到记录（26/30=86.7% < 90%）

**操作**：
- 不做任何交易，保持当前持仓不变
- 权重数据质量不足，计算结果不可靠

---

### 场景 9：成分权重保有率充足但部分缺失

**条件**：
- 入选的30只股票中，28只有权重数据（28/30=93.3% ≥ 90%）
- 2只股票找不到成分权重记录

**操作**：
- 28只股票使用指数真实成分股权重
- 2只缺失权重数据的股票使用等权 `1/n` 作为 fallback 基准权重
- 归一化保证最终权重之和 = 100%，不会超限

**fallback 的影响**：
- `1/n` 是一个中间值 fallback，可能大于或小于真实权重
- 但缺失权重是少数边缘情况，且归一化会均摊误差，对整体组合影响有限

***

## 核心业务逻辑澄清

### 权重计算流程

1. **数据质量检查**：
   - 因子分数保有率 < `min_score_coverage` → 不操作
   - 缺失因子的股票不参与排名
2. **股票选择**：从有效因子分数的成分股中，选择排名前 `holding_count` 的股票
3. **权重质量检查**：
   - 成分权重保有率 < `min_weight_coverage` → 不操作
   - 缺失权重的股票使用等权 `1/n` 作为 fallback
4. **基准权重**：读取入选股票的指数真实成分股权重
5. **倾斜计算**：`tilt = z × active_weight_bound`，然后 clip 到 `[-bound, +bound]`
6. **最终权重**：`权重 = 基准权重 + tilt`，clip 到非负，再归一化使总权重为 1

## 已确认的业务逻辑

1. **因子分数保有率检查**：保有率 < `min_score_coverage` 时本调仓日不操作
2. **成分权重保有率检查**：保有率 < `min_weight_coverage` 时本调仓日不操作
3. **基准权重**：使用指数真实成分股权重，而非等权
4. **选股规则**：按因子得分排名前 `holding_count`，无缓冲区
5. **权重约束**：仅 `active_weight_bound` 单票偏离约束（风控作用），无组合层面约束
6. **因子数据缺失处理**：因子得分缺失的股票不参与排名，不可能入选持仓
7. **基准权重缺失处理**：使用等权 `1/n` 作为 fallback
8. **归一化**：非入选股票权重归零，入选股票权重归一化至 100%

## 最终业务逻辑总结

1. **因子分数保有率**：保有率 < `min_score_coverage` 时本调仓日不操作
2. **成分权重保有率**：保有率 < `min_weight_coverage` 时本调仓日不操作
3. **股票选择**：从有效因子分数的成分股中买入排名前 `holding_count` 的股票（无缓冲机制）
4. **基准权重**：使用指数真实成分股权重
5. **权重倾斜**：z-score × active_weight_bound，clip 约束单票偏离（风控作用）
6. **归一化**：最终权重归一化至 100%
7. **因子缺失**：因子得分缺失的股票不参与排名，不可能入选
8. **基准权重缺失**：使用等权 `1/n` 作为 fallback
9. **调仓日独立性**：每个调仓日独立计算
10. **交易约束**：停牌跳过不延续、涨停跳过不延续、跌停跳过但后续每日重试直到下一个调仓日（详见 [trade-constraints-design.md](trade-constraints-design.md)）
