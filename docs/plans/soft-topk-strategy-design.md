# SoftTopK 策略业务场景设计

## 当前配置参数

```yaml
SoftTopK:
  holding_count: 30       # 持仓股票数（按因子分从高到低选前N只）
  weight_func: softmax    # 权重函数：softmax 或 rank_power
  softmax_temperature: 1.0  # softmax 温度参数（T 越小集中度越高）
  rank_power_alpha: 1.0    # rank_power 衰减参数（α 越大头部越集中）
  min_score_coverage: 0.90  # 因子分数保有率阈值（低于此值本调仓日不操作）
```

## 策略概述

**SoftTopK（软性TopK策略）** 的核心逻辑：

1. **选股**：从股票池中，按因子得分从高到低排序，选前 `holding_count` 只，因子缺失的股票不参与排名
2. **权重分配**：根据因子得分连续分配权重（而非简单等权），使"高置信度信号"获得更高仓位
3. **无缓冲机制**：不考虑当前持仓，每个调仓日全量重算

## 核心参数说明

### `holding_count` - 持仓股票数

**含义**：按因子得分从高到低选前 N 只股票持仓。

**影响**：
- N 越大（如 50~80）：越分散，头部集中度越低
- N 越小（如 15~20）：越集中，头部股票权重更大

### `weight_func` - 权重函数

支持两种权重分配方式：

#### `softmax`（默认）

- **公式**：`weight_i = exp(score_i / T) / Σ exp(score_j / T)`
- **特点**：对高分更敏感，头部集中度更高
- **温度参数** `softmax_temperature`（默认 1.0）：
  - T 越小：集中度越高（接近赢家通吃）
  - T 越大：越接近等权
- **直觉**：得分差异越大，权重差异越大

#### `rank_power`

- **公式**：`weight_i = 1 / rank_i^α`
- **特点**：按排名幂律衰减，权重更平滑、稳健
- **衰减参数** `rank_power_alpha`（默认 1.0）：
  - α = 1：标准调和衰减
  - α > 1：衰减更快，头部更集中
  - α < 1：衰减更慢，更接近等权
- **直觉**：不依赖原始分数值，只依赖排名，对异常值更鲁棒

### `min_score_coverage` - 因子分数保有率阈值

**含义**：调仓日因子分数非缺失的股票数占股票池总数的最低比例。

**作用**：
- 若保有率 < `min_score_coverage`，本调仓日不做任何操作
- 若保有率 ≥ `min_score_coverage`，缺失因子的股票不参与排名

## 核心业务逻辑澄清

### 权重计算流程

1. **数据质量检查**：因子分数保有率 < `min_score_coverage` → 不操作
2. **股票选择**：按因子得分排名取前 `holding_count` 只
3. **权重计算**：按 `weight_func`（softmax 或 rank_power）计算连续权重

### 两种权重函数对比

| 维度 | softmax | rank_power |
|------|---------|------------|
| 依赖原始分数值 | 是 | 否（仅依赖排名） |
| 对异常值敏感度 | 较高 | 较低 |
| 集中度可调性 | 通过温度参数 | 通过衰减参数 |
| 适用场景 | 因子得分分布均匀、差异可信 | 因子得分波动大、排名更可靠 |

## 已确认的业务逻辑

1. **因子分数保有率检查**：保有率 < `min_score_coverage` 时本调仓日不操作
2. **选股规则**：按因子得分排名前 `holding_count`，无缓冲区
3. **权重分配**：连续分配（非等权），高分股票权重更大
4. **因子数据缺失处理**：因子得分缺失的股票不参与排名
5. **停牌、涨停、跌停处理**：信号日（调仓日）当天不用关注停牌、涨停、跌停股票，因为信号日也不用交易，只用基于因子得分进行排名就行。信号日+1天用开盘价进行交易，停牌直接跳过(不延续)，买入跳过涨停（不延续）、卖出跳过跌停（后续每日用开盘价重试卖出直到下一个调仓日，重试卖出以开盘价执行，如果开盘时不跌停就可以卖出）。因为存在停牌、涨停、跌停导致交易失败的情况，所以每个调仓日之后持股数量和holding_count会有差距，这个是正常的。

```yaml
Execution:
  suspend_action: skip          # 停牌处理：skip（跳过，不延续）
  limit_up_action: skip_buy     # 涨停处理：skip_buy（跳过买入，不延续）
  limit_down_action: delay_sell # 跌停处理：delay_sell（跳过卖出，后续每日重试直到下一个调仓日）
```

6. SoftTopK存在"部分卖出"的情况：
- 某股当前权重6%，目标权重3% → 需要卖出一半
- 如果该股跌停 → 跳过，后续每日重试
跌停重试时，是卖出"目标权重与当前权重的差额"
7. 涨停/停牌跳过后的替补规则 ：跳过的股票不替补，持仓数可能少于holding_count，这是正常的。因为SoftTopK的权重是按因子分数精确分配的，替补排名靠后的股票会稀释整体权重分配的合理性。
8. 假设holding_count=30，如果2只涨停跳过，剩余28只的权重怎么算？剩余28只仍按原始softmax/rank_power权重分配（权重之和<1，现金比例增加）
9. **调仓日独立性**：每个调仓日独立计算信号，不考虑前序调仓日的遗留交易（停牌、涨停、跌停而导致的交易失败）
10. 现金缓冲 ：保留cash_buffer_ratio的现金缓冲，防止因滑点导致资金不足，导致交易失败
11. 权重再平衡方式 ：每个调仓日全量重算目标权重，所有持仓调整到目标权重（包括部分买入和部分卖出）。与TopKDropout的方式B不同，SoftTopK允许部分买入/部分卖出的调整交易。
