# AI量化因子挖掘系统 - 工程师实操手册

本文档为基于大模型 (LLM) 的全自动量化选股因子挖掘系统的技术实操指南。

## 1. 环境与前置准备

### 1.1 创建 Python 环境

首次在 PowerShell 中使用 Conda 时，需先进行初始化：

```powershell
& D:\miniconda3\Scripts\conda.exe init powershell
```

关闭并重新打开终端后，执行以下命令创建并激活环境：

```powershell
conda create -n factor-factory python=3.13 -y
conda activate factor-factory
python -V
```

### 1.2 安装项目依赖

项目依赖已统一整理至项目根目录的 `requirements.txt` 文件中。在激活 `factor-factory` 环境后，可以通过以下命令一键安装所有依赖：

```powershell
pip install --upgrade pip
pip install -r requirements.txt -i http://mirrors.aliyun.com/pypi/simple/ --trusted-host mirrors.aliyun.com
```

**核心依赖包说明：**

- **数据与处理**：`tushare` (数据获取)、`pandas` / `numpy` (特征处理)、`pyarrow` (Parquet 读写)
- **大模型调用**：`openai` (API 接口)、`httpx` (HTTP 客户端)、`pydantic` (格式校验)
- **回测与分析**：`alphalens-reloaded` (OOS 因子复盘)
- **数据分析与可视化**：`scipy` / `scikit-learn` (辅助分析)、`matplotlib` (绘图)、`jupyterlab` (交互式分析环境)
- **工具与看板**：`PyYAML` (配置读取)、`python-dotenv` (环境变量)、`streamlit` (图形看板)

### 1.3 配置文件与参数准备

在运行代码前，请确保已完成以下基础配置：

1. **环境变量**：复制 `runtime/config/env.yaml.example` 并重命名为 `runtime/config/env.yaml`，填入必需的 Token（如 Tushare Token, LLM API Key）。
2. **运行模式**：检查 `runtime/config/analysis_rule.yaml` 中的 `run_mode` 是否符合当前预期阶段（`train` 样本内训练 或 `test` 样本外盲测）。

### 1.4 核心执行入口与步骤

在 `runtime` 目录下，按以下顺序或需求执行相应脚本：

- **训练窗口推荐**（独立前置）：
  ```powershell
  python src/train_window_selector.py
  ```
  *(读取* *`config/selector.yaml`，评估多种窗口长度，生成推荐报告)*
- **样本内训练主流程**（需设置 `run_mode: train`）：
  ```powershell
  python src/step10_iterate.py
  ```
- **样本外盲测主流程**（需设置 `run_mode: test`）：
  ```powershell
  python src/step11_oos_test.py
  ```
- **OOS 因子复盘**（Alphalens 分析）：
  ```powershell
  python src/step12_alphalens_report.py
  ```
  *(基于* *`outputs/backtest/top3_factors.json`* *进行图表和指标分析)*
- **OOS 图形看板**（Streamlit 交互式查看）：
  ```powershell
  cd d:\test\factor-factory\runtime
  streamlit run src/step13_alphalens_dashboard.py --server.port 8501 --server.address 127.0.0.1 --server.headless true
  ```
  *(启动后可在浏览器点开即看)*
- **生成聚宽回测代码**：
  ```powershell
  python src/step14_joinquant_export.py --factor <因子名>
  ```
  *(读取指定优秀因子，生成可直接粘贴至 JoinQuant 运行的策略代码)*

### 1.5 Step10 / Step11 执行流程总表

下面这张表只回答一个核心问题：`Step10` 和 `Step11` 到底按什么顺序执行、哪些步骤只跑一次、哪些步骤会重复跑。

#### Step10（样本内训练）执行流程

| 层级           | 阶段         | 执行内容                                                                 | 执行次数              | 主要输出 / 作用                                                                                               |
| ------------ | ---------- | -------------------------------------------------------------------- | ----------------- | ------------------------------------------------------------------------------------------------------- |
| 全局           | 初始化        | `Step00` 清理输出目录                                                      | 整个 `step10` 只 1 次 | 清空旧的 `outputs` 运行产物，避免污染本轮训练                                                                            |
| 全局           | 预缓存        | `Step01` 预缓存 Tushare 数据                                              | 整个 `step10` 只 1 次 | 提前准备 train/test 所需底层数据缓存，减少后续等待                                                                         |
| 每个 window    | 切换上下文      | 设置当前 `window` 的 discovery 上下文                                        | 每个 `window` 1 次   | 后续步骤都基于该窗口的 `discovery_start_date ~ discovery_end_date` 运行                                              |
| 每个 window    | 特征体检       | `Step02` 特征健康检查                                                      | 每个 `window` 1 次   | 生成 `feature_stats.csv`、`feature_corr.csv`、`health_summary.json`                                         |
| 每个 window    | 市场分析       | `Step03` 市场环境分析                                                      | 每个 `window` 1 次   | 生成 `market_context.json`，给 LLM 提供当前窗口市场状态                                                               |
| 每个 iteration | 摘要构建       | `Step04` 构建 LLM 摘要                                                   | 每轮迭代 1 次          | 汇总 `Step02/03` 结果，并可带入上一轮反馈                                                                             |
| 每个 iteration | 候选生成       | `Step05` 调用 LLM                                                      | 每轮迭代 1 次          | 生成本轮候选因子原始输出                                                                                            |
| 每个 iteration | 公式校验       | `Step06` 校验因子公式                                                      | 每轮迭代 1 次          | 生成 `factors_validated.json`、`factors_rejected.json`                                                     |
| 每个 iteration | 因子评估       | `Step07` 计算 IC / RankIC / IR                                         | 每轮迭代 1 次          | 生成 `factor_values.parquet`、`factor_metrics.csv`                                                         |
| 每个 iteration | 回测硬筛       | `Step08` 回测与硬筛（mean\_rank\_ic，rank\_ic\_ir，win\_rate，llm\_direction） | 每轮迭代 1 次          | 生成 `strategy_metrics.csv`、`skipped_factors.json` 等                                                      |
| 每个 iteration | 综合打分       | `Step09` 综合打分                                                        | 每轮迭代 1 次          | 生成 `final_score.csv`、`top3_factors.json`                                                                |
| 每个 iteration | 结果归档       | 归档本轮 discovery 产物                                                    | 每轮迭代 1 次          | 写入 `outputs/train_windows/<window_id>/discovery/iter_XX/`                                               |
| 每个 window    | 候选汇总       | 汇总 discovery 各轮候选并按 `formula` 去重                                     | 每个 `window` 1 次   | 保留该窗口里表现最好的候选公式（同公式仅保留 Step09 total\_score 最高的一条记录）                                                     |
| 每个 window    | validation | 进入 validation，只跑 `Step07~Step09`                                     | 每个 `window` 1 次   | 检验 discovery 候选在 validation 区间是否仍能通过硬筛（mean\_rank\_ic，rank\_ic\_ir，win\_rate，llm\_direction）并保持可接受的综合表现 |
| 全局           | 跨窗口汇总      | 汇总全部窗口 validation 结果                                                 | 整个 `step10` 只 1 次 | 生成跨窗口总榜与最终训练结论                                                                                          |

#### Step10（样本内训练）简化时序图

```text
Step00
  -> Step01
  -> for each window
       -> set discovery context
       -> Step02
       -> Step03
       -> for iteration = 1..N
            -> Step04
            -> Step05
            -> Step06
            -> Step07
            -> Step08
            -> Step09
            -> archive iter outputs
       -> collect discovery candidates
       -> validation: Step07 -> Step08 -> Step09
  -> cross-window aggregation
```

#### Step11（样本外盲测）执行流程

| 层级 | 阶段   | 执行内容                    | 执行次数              | 主要输出 / 作用                                                       |
| -- | ---- | ----------------------- | ----------------- | --------------------------------------------------------------- |
| 全局 | 初始化  | 清理运行上下文                 | 整个 `step11` 只 1 次 | 确保当前流程处于干净的 test 上下文                                            |
| 全局 | 预缓存  | `Step01` 预缓存 Tushare 数据 | 整个 `step11` 只 1 次 | 准备 test 区间所需缓存                                                  |
| 全局 | 候选收集 | 收集训练阶段优胜因子              | 整个 `step11` 只 1 次 | 来源优先为 `cross_window_top3_factors.json`，其次回退到各窗口 validation Top3 |
| 全局 | 因子评估 | `Step07` 计算 OOS 指标      | 整个 `step11` 只 1 次 | 在 test 区间计算因子统计表现                                               |
| 全局 | 回测检验 | `Step08` 样本外回测          | 整个 `step11` 只 1 次 | 评估样本外收益、回撤、换手等表现                                                |
| 全局 | 综合打分 | `Step09` OOS 综合打分       | 整个 `step11` 只 1 次 | 输出最终 OOS 排名和 Top3                                               |

#### Step11（样本外盲测）简化时序图

```text
Step01
  -> collect train top factors
  -> Step07
  -> Step08
  -> Step09
```

#### Step10 与 Step11 的一句话区别

| 流程       | 是否生成新因子 | 是否有多轮 iteration | 是否有 validation | 最终目标                       |
| -------- | ------- | --------------- | -------------- | -------------------------- |
| `Step10` | 是       | 是               | 是              | 在 train 区间完成海选、复验、跨窗口总榜    |
| `Step11` | 否       | 否               | 否              | 在 test 区间对训练优胜因子做真正 OOS 检验 |

## 5. 股票池管理指南

### 5.1 概述

`stock_pool_manager.py` 是系统的股票池入口组件。\
当前版本数据源为 `tushare-only`，股票池逻辑都基于 `TushareProvider` 获取标的。

### 5.2 核心组件架构

```
stock_pool_manager.py  # 统一的股票池管理器
├── 支持多种股票池类型
│   ├── all_market: 全市场
│   ├── index_components: 指数成分股
│   ├── industry: 行业股票池
│   ├── custom_list: 自定义列表
│   ├── random_sample: 随机抽样
│   └── stratified_sample: 分层抽样
└── 当前数据源
    └── TushareProvider
```

### 5.3 配置方式

在 `env.yaml` 中配置股票池：

```yaml
stock_pool:
  type: "index_components"  # 股票池类型
  index_code: "000300.XSHG"  # 沪深300
```

常用附加过滤参数：

```yaml
stock_pool:
  include_st: false
  include_new_stock: false
  new_stock_days: 60
```

- `include_st`: 是否保留 ST 股票
- `include_new_stock`: 是否保留上市未满 `new_stock_days` 的新股

### 5.4 支持的股票池类型

#### 类型汇总

| 类型                  | 说明      | 适用场景      | 支持状态         |
| ------------------- | ------- | --------- | ------------ |
| `all_market`        | 全市场股票池  | 全市场因子挖掘   | ✅ 可用         |
| `index_components`  | 指数成分股   | 生产环境、风格控制 | ✅ 可用         |
| `industry`          | 行业股票池   | 行业内研究     | ⚠️ 依赖数据源行业接口 |
| `custom_list`       | 自定义股票列表 | 固定研究池     | ✅ 可用         |
| `random_sample`     | 随机抽样    | 快速验证、开发调试 | ✅ 可用         |
| `stratified_sample` | 分层抽样    | 控制样本代表性   | ✅ 可用（简化实现）   |

#### 1. 全市场股票池

```yaml
stock_pool:
  type: "all_market"
```

**适用场景**：全市场因子挖掘和基准实验

#### 2. 指数成分股（推荐）

```yaml
stock_pool:
  type: "index_components"
  index_code: "000300.XSHG"  # 沪深300
```

**适用场景**：策略落地时更稳定的可交易池

**常用指数代码（系统统一格式）**：

| 指数名称   | 配置值                        |
| ------ | -------------------------- |
| 沪深300  | `000300.XSHG` 或 `SH000300` |
| 中证500  | `000905.XSHG` 或 `SH000905` |
| 中证1000 | `000852.XSHG` 或 `SH000852` |
| 上证50   | `000016.XSHG` 或 `SH000016` |
| 深证成指   | `399001.XSHE` 或 `SZ399001` |
| 创业板指   | `399006.XSHE` 或 `SZ399006` |

系统会在数据提供层自动映射成 Tushare 接口需要的格式（如 `000300.SH`）。

#### 3. 行业股票池

```yaml
stock_pool:
  type: "industry"
  industry_name: "银行"
  industry_level: 1
```

**适用场景**：行业内选股研究\
**注意**：是否可用取决于数据源对行业分类接口的支持

#### 4. 自定义股票列表

```yaml
stock_pool:
  type: "custom_list"
  instruments:
    - "000001.XSHE"
    - "600000.XSHG"
```

**适用场景**：生产环境固定池、白名单池

#### 5. 随机抽样

```yaml
stock_pool:
  type: "random_sample"
  base_pool_type: "all_market"
  sample_size: 100
  seed: 42  # 固定种子，保证可复现
```

**适用场景**：快速验证系统性能和链路连通性

#### 6. 分层抽样

```yaml
stock_pool:
  type: "stratified_sample"
  base_pool_type: "all_market"
  sample_size: 500
  stratify_by: ["market_cap"]
  seed: 42
```

**适用场景**：保证样本代表性\
**注意**：当前实现是“简化分层”，默认按代码排序分层近似处理，不是严格市值分位分层
