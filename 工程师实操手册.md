# AI量化因子挖掘系统 - 工程师实操手册

本文档为基于 Qlib + 大模型 (LLM) 的全自动量化选股因子挖掘系统的技术实操指南。专为开发/投研人员设计，聚焦于"怎么配、怎么跑、结果怎么看"。

---

## 1. 环境与前置准备

### 1.1 创建 Python 环境

第一次在 PowerShell 里用 conda 的话，先初始化一次：

```powershell
& D:\miniconda3\Scripts\conda.exe init powershell
```

关掉终端，重新打开后，再执行：

```powershell
conda create -n qlib_v1 python=3.8 -y
conda activate qlib_v1
python -V
```

这里建议先用 Python 3.8，是为了尽量贴近 Qlib 官方文档在 Windows 下更稳的使用方式。

---

### 1.2 安装依赖包

激活 `qlib_v1` 环境后，先安装这些包：

```powershell
pip install --upgrade pip
pip install tushare rqdatac pyqlib lightgbm torch pandas numpy scipy scikit-learn pyarrow openai pydantic python-dotenv mlflow jupyterlab matplotlib
```

这批包的作用大概是：

- `tushare`：Tushare 数据客户端
- `rqdatac`：米筐数据客户端
- `pyqlib`：Qlib 主体
- `lightgbm`、`torch`：Qlib 常用依赖
- `pandas`、`numpy`：做特征处理
- `openai`：调用大模型接口
- `pydantic`：校验大模型输出格式
- `mlflow`：保存实验记录

### 1.5 配置大模型 API

系统强依赖大模型（如 OpenAI、DeepSeek 等），请确保在运行前已设置环境变量。

如果你用的是 DeepSeek，由于 Windows 下 `setx` 命令设置后**需要重启终端才能生效**，建议你直接在当前的 PowerShell 里用 `$env:` 的方式设置（这样当前窗口立刻生效）：

```powershell
$env:OPENAI_API_KEY="sk-你的真实密钥"
$env:OPENAI_BASE_URL="https://api.deepseek.com"
$env:OPENAI_MODEL="deepseek-reasoner"
```

如果你希望永久保存（下次打开终端自动生效），也可以执行 `setx`：

```powershell
setx OPENAI_API_KEY "sk-你的真实密钥"
setx OPENAI_BASE_URL "https://api.deepseek.com"
setx OPENAI_MODEL "deepseek-reasoner"
```

> **注意**：执行完 `setx` 后，当前终端依然读不到新变量，**必须关闭当前终端并重新打开**才能生效！

第一版系统里，我更建议先用 `deepseek-chat`，因为它更适合稳定输出结构化 JSON。

设置完以后，重开一个终端，再验证：

```powershell
python -c "import os; print(os.getenv('OPENAI_MODEL'))"
```

看到模型名能正常打印，就说明环境变量生效了。

## 2. 核心配置文件 (`env.yaml`)

所有的核心运行逻辑都在 `config/env.yaml` 中控制。**每次运行前，请务必检查此文件！**

关键配置项解读：
```yaml
# 核心运行模式（极其重要！）
# 'train': 样本内训练模式。大模型会根据表现不断迭代生成新因子。
# 'test': 样本外盲测模式。不再调用大模型，直接用训练集选出的 Top3 因子在全新时间段跑回测。
run_mode: train

# 样本内 (In-Sample) 训练时间段
train_start_date: 2018-01-01
train_end_date: 2022-12-31

# 样本外 (Out-of-Sample) 盲测时间段
test_start_date: 2023-01-01
test_end_date: 2024-05-20

# 因子筛选严格标准 (Step 08 使用)
min_rank_ic_to_backtest: 0.015
min_rank_ic_ir_to_backtest: 0.15
min_positive_ic_ratio: 0.45
enable_direction_filter: false   # 是否额外要求 llm_direction 与 empirical_direction 一致
```

## 3. 标准作业流程 (SOP)

整个系统的工作流被设计为**"清理环境 -> 样本内训练 -> 样本外盲测 -> 第三方平台因子检验 -> 第三方平台策略回测"**五阶段模式，以确保因子的有效性和跨平台一致性。

### 阶段零：清理环境 (Clean Environment)

**目标**：在开始新一轮因子挖掘前，清理上一轮运行产生的临时文件，确保环境干净。

**操作步骤**：
1. 打开终端，进入 `runtime` 目录。
2. 执行清理脚本：
   ```bash
   # 预览模式（只查看将要删除的文件，不实际删除）
   python src/step00_clean.py --dry-run
   
   # 实际清理
   python src/step00_clean.py
   ```

**清理范围**：
- `outputs/health/` - 健康检查生成的统计文件
- `outputs/llm/` - LLM 生成的因子文件（验证通过和拒绝的）
- `outputs/backtest/` - 回测结果文件
- `outputs/iter_*/` - 历次迭代的中间文件

**注意事项**：
- 清理脚本会保留 `outputs/` 目录结构，只删除其中的文件
- 如果某些文件被其他程序占用（如 Excel 打开 CSV 文件），脚本会跳过这些文件并提示
- 建议在开始新一轮完整实验前执行清理，避免旧数据干扰

### 阶段一：样本内训练与挖掘 (In-Sample Training)

**目标**：让大模型在样本内训练数据上反复试错，挖掘出具有强超额收益的 Alpha 因子。

**操作步骤**：
1. 打开 `config/env.yaml`，确保设置：
   - `run_mode: train`
   - `train_start_date` 和 `train_end_date`（如 2018-01-01 至 2020-12-31）
   - `iteration_count`（迭代次数，如 3）
2. 打开终端，进入 `runtime` 目录。
3. 执行一键迭代脚本：
   ```bash
   python src/step10_iterate.py
   ```
4. 脚本会自动根据 `env.yaml` 中的 `iteration_count` 参数进行迭代（例如设置为 3，表示让大模型进化 3 代）。

**产出文件与解读**：
脚本运行完毕后，会在 `outputs/` 下生成 `iter_01`, `iter_02`, `iter_03` 目录。
每个 `iter_XX` 目录中你需要重点关注：
- `backtest/top3_factors.json`: **最核心的产出**。这是本轮迭代中综合打分（收益率、回撤、IC稳定性等加权）最高的 3 个因子，包含公式、`llm_direction`（大模型原始猜测）和 `empirical_direction`（样本内经验方向）等详细指标。
- `backtest/strategy_metrics.csv`: 这 3 个因子组合回测的详细财务指标。
- `backtest/skipped_factors.json`: 被系统残酷淘汰的因子名单及死因（IC太低/回撤太大等）。这会被喂给下一轮大模型作为"反面教材"。

### 阶段二：样本外盲测 (Out-of-Sample Test)

**目标**：检验阶段一中选出来的"好学生"，在他们完全没见过的 2023-2024 年数据上，是否依然能赚钱（检验是否过拟合）。

**操作步骤**：
1. 打开 `config/env.yaml`，将 `run_mode` 修改为 `test`。
2. 打开终端，执行盲测脚本：
   ```bash
   python src/step11_oos_test.py
   ```

**执行逻辑**：
脚本会自动去扫描所有的 `iter_XX/backtest/top3_factors.json`，把里面所有去重后的优秀因子公式提取出来，并使用其中记录的 `empirical_direction` 作为样本外盲测方向。`top3_factors.json` 的标准方向字段只有 `llm_direction` 和 `empirical_direction`。**此阶段绝对不调用大模型**。

**产出文件与解读**：
执行结束后，去 `outputs/backtest/` 目录查看：
- `top3_factors.json`: 在盲测数据上，表现最好的那几个因子。如果这里的因子和训练集表现差异极大，说明因子已经失效或过拟合；如果表现依然坚挺，说明你挖到了真正的"跨周期 Alpha"！
- `strategy_metrics.csv`: 盲测期内的真实年化收益率、最大回撤等硬核数据。

### 阶段三：第三方平台因子检验 (Factor Validation)

**目标**：在米筐等第三方量化平台上验证因子的 IC、ICIR 等指标，确保因子逻辑在不同数据源下依然有效。

**操作步骤**：
1. 生成米筐因子代码：
   ```bash
   python ricequant/factor_converter.py
   ```
2. 登录米筐平台，进入"因子研究" → "因子检验"。
3. 复制生成的 `.py` 文件内容到因子编辑器。
4. 设置检验参数（时间范围、股票池、基准等）。
5. 点击"运行检验"查看 IC、ICIR 等指标。

**验证标准**：
- 米筐平台的 IC、ICIR 与 Qlib 结果差异 < 0.02
- 因子方向一致（正相关/负相关）

**产出文件**：
- `ricequant/converted_factors/{factor_name}.py`: 米筐因子代码

### 阶段四：第三方平台策略回测 (Strategy Backtest)

**目标**：在米筐等第三方量化平台上进行完整的策略回测，验证策略的实际收益表现。

**操作步骤**：
1. 生成米筐策略代码：
   ```bash
   python ricequant/validator.py
   ```
2. 登录米筐平台，进入"策略回测"页面。
3. 复制生成的策略代码到左侧编辑器。
4. 在右侧界面设置回测参数（日期、资金、基准、调仓频率等）。
5. 点击"运行回测"开始验证。

**验证标准**：
- 年化收益差异 < 5%
- 最大回撤差异 < 3%
- Sharpe 比率差异 < 0.2

**产出文件**：
- `ricequant/strategies/{factor_name}_strategy.py`: 米筐策略代码
- `ricequant/strategies/strategy_summary.md`: 策略汇总报告

## 4. 各子步骤拆解 (单步调试指南)

如果你不想一键运行，或者在某个环节报错需要排查，你可以单独运行 `step01` 到 `step09`。以下是基于源码真实逻辑的每一步详细说明（包含确切的输入、执行动作、产出文件及解读）：

### Step 00: 清理临时文件 (`step00_clean.py`)
- **输入**: 无（直接执行）。
- **动作**: 在开始新的因子挖掘流程前，清理 `outputs/` 目录中的临时内容，确保从一个干净的状态开始。
- **清理内容**:
  - `health/` 目录下的特征体检文件
  - `iter_*/` 目录下的迭代中间文件
  - `backtest/` 目录下的回测结果文件
  - `llm/` 目录下的大模型输出文件
- **保留内容**: `config/` 和 `data/` 目录（配置文件和数据源）。
- **使用方式**:
  ```bash
  # 预览模式（查看将要删除的内容，不实际删除）
  python src/step00_clean.py --dry-run
  
  # 实际清理
  python src/step00_clean.py
  ```

### Step 01: 初始化数据源 (`step01_init_datasource.py`)
- **输入**: `config/env.yaml` (读取数据源类型、时间范围、数据路径等配置)。
- **动作**: 根据配置初始化数据源（Qlib、Tushare 或米筐），获取股票列表和样本数据预览。
- **输出**: `outputs/health/sample_preview.csv` (包含前 50 行的样本股票数据预览，确认数据加载成功)。

### Step 02: 构建特征池 (`step02_build_feature_pool.py`)
- **输入**: `config/feature_pool.yaml`。
- **动作**: 读取特征池配置文件，提取系统支持的基础特征和算子列表。
- **输出**: `outputs/health/feature_pool_manifest.json` (一份打包好的 JSON 清单，包含了所有允许大模型使用的基础特征表达式和对应名称)。

### Step 03: 基础体检 (`step03_health_check.py`)
- **输入**: `config/env.yaml`, `config/feature_pool.yaml`，以及底层加载的股票行情数据。
- **动作**: 计算特征池中各项指标的缺失值统计、极值检查，并计算各特征间的自相关系数（防止重复冗余）。
- **输出**:
  - `outputs/health/feature_stats.csv`: 单个特征的质量报告（包含缺失率、均值、极值等）。
  - `outputs/health/feature_corr.csv`: 特征间的相关系数矩阵。
  - `outputs/health/health_summary.json`: 对上述体检结果的初步 JSON 摘要。

### Step 04: 构建摘要 (`step04_build_summary.py`)
- **输入**: `outputs/health/feature_stats.csv`, `outputs/health/feature_corr.csv`，以及上轮迭代产生的 `outputs/backtest/top3_factors.json` (如果存在)。
- **动作**: 将 Step 03 的统计 CSV 进一步压缩提炼为大模型容易理解的 JSON 摘要，并附加上一轮的成功因子经验。
- **输出**: `outputs/health/llm_summary.json` (发送给大模型的最核心的提示词素材)。

### Step 05: 调用大模型 (`step05_call_llm.py`)
- **输入**: `config/env.yaml`, `config/feature_pool.yaml`, `config/backtest_rule.yaml`, `outputs/health/llm_summary.json`，以及上轮产生的 `outputs/backtest/top3_factors.json` 和 `outputs/backtest/skipped_factors.json` (如果存在)。外加环境变量中的大模型 API 密钥。
- **动作**: 组装完整的 Prompt 请求大模型生成新因子，并在代码内强制关闭网络代理以免 VPN 阻断连接。
- **输出**: `outputs/llm/raw_response.json` (大模型返回的原始字符串结果和请求元数据，未经任何处理)。

### Step 06: 公式校验 (`step06_validate_factor.py`)
- **输入**: `outputs/llm/raw_response.json`, `config/feature_pool.yaml`, `config/backtest_rule.yaml`。
- **动作**: 尝试解析大模型的 JSON 输出，拦截有语法错误、使用了未授权字段、或者私自篡改回测规则的非法因子。这里仍要求大模型返回 `direction`，但它在通过校验后会被标准化保存为 `llm_direction`，用于表示“大模型的原始方向假设”。
- **输出**:
  - `outputs/llm/factors_validated.json`: 语法正确、合法且通过校验的因子列表，其中方向字段会统一命名为 `llm_direction`。
  - `outputs/llm/factors_rejected.json`: 被判定为非法的因子列表及其具体的拒绝原因报错信息。

### Step 07: 因子评估 (`step07_eval_factor.py`)
- **输入**: `config/env.yaml`, `config/feature_pool.yaml`, `outputs/llm/factors_validated.json`，以及底层行情数据。
- **动作**: 让 Qlib 真正加载这些合法的公式，直接计算 `raw_score`，并基于原始分数得出 IC (信息系数) 等初步纯因子指标；同时把 LLM 提供的 `llm_direction` 与样本内计算得到的 `empirical_direction` 一并写入指标表。
- **输出**:
  - `outputs/backtest/factor_values.parquet`: 包含每个股票在每个交易日的实际因子计算数值（体积较大）。
  - `outputs/backtest/factor_metrics.csv`: 每个因子的初步统计指标（IC均值、ICIR、正向胜率、`llm_direction`、`empirical_direction` 等）。

### Step 08: 策略回测 (`step08_backtest.py`)
- **输入**: `config/env.yaml`, `config/feature_pool.yaml`, `config/backtest_rule.yaml`, `outputs/backtest/factor_values.parquet`, `outputs/backtest/factor_metrics.csv`。
- **动作**: 首先根据 `env.yaml` 设定的阈值淘汰 IC 或 ICIR 不达标的因子，然后仅根据样本内数据推导出的 `empirical_direction` 决定回测时是否翻转分数；若 `enable_direction_filter: true`，则还会额外要求 `llm_direction` 与 `empirical_direction` 一致。这是**最耗时**的一步。
- **输出**:
  - `outputs/backtest/strategy_metrics.csv`: 幸存因子的详细回测财务指标（年化收益、最大回撤、夏普比率等）。
  - `outputs/backtest/skipped_factors.json`: 被淘汰的弱效因子名单及具体死因。
  - `outputs/backtest/positions.parquet`: 模拟交易的每日持仓明细。
  - `outputs/backtest/orders.parquet`: 模拟交易的买卖订单流水。
  - `outputs/backtest/nav_curve.parquet`: 净值曲线数据。

### Step 09: 综合打分 (`step09_score.py`)
- **输入**: `outputs/backtest/factor_metrics.csv`, `outputs/backtest/strategy_metrics.csv`, `outputs/llm/factors_validated.json`。
- **动作**: 综合各项指标计算总分，提高收益率权重，对高回撤、高换手和负收益因子进行严重扣分惩罚。最终按总分排序选出最强的 Top 3。
- **输出**:
  - `outputs/backtest/final_score.csv`: 所有进入回测因子的综合打分明细。
  - `outputs/backtest/top3_factors.json`: 最终选出的前三名优秀因子的全量信息，包含 `llm_direction` 与 `empirical_direction` 两套方向信息（会被移入 `iter_XX` 存档，或作为下轮迭代的依据）。

---
**工程师调试贴士**：
如果你修改了底层的评分逻辑（比如改了 `step09` 里的权重），你不需要重新跑大模型，只需要重新运行 `python src/step09_score.py` 即可立刻看到新的打分结果。

## 5. 股票池管理指南

### 5.1 概述

`stock_pool_manager.py` 是系统的核心组件，提供统一的股票池管理能力。它支持多种股票池类型，确保在不同数据源（Qlib、米筐、聚宽等）下都能灵活配置股票池。

### 5.2 核心组件架构

```
stock_pool_manager.py  # 统一的股票池管理器
├── 支持多种股票池类型
│   ├── all_market: 全市场
│   ├── index_components: 指数成分股
│   ├── industry: 行业股票池
│   ├── custom_list: 自定义列表
│   ├── random_sample: 随机抽样
│   └── stratified_sample: 分层抽样
└── 跨平台兼容
    ├── Qlib: 当前数据源
    ├── 米筐: 未来数据源
    ├── 聚宽: 未来数据源
    └── Tushare: 未来数据源
```

### 5.3 配置方式

在 `env.yaml` 中配置股票池：

```yaml
stock_pool:
  type: "index_components"  # 股票池类型
  index_code: "000300.XSHG"  # 沪深300
```

更多配置示例参考 `runtime/config/env_stock_pool_examples.yaml`。

### 5.4 支持的股票池类型

#### 类型汇总

| 类型 | 说明 | 适用场景 | 支持状态 |
|------|------|----------|----------|
| `all_market` | 全市场股票池 | 全市场因子挖掘 | ✅ 完全支持 |
| `index_components` | 指数成分股 | 跨平台验证、生产环境 | ✅ 完全支持 |
| `industry` | 行业股票池 | 特定行业研究 | ⚠️ 需要数据源支持 |
| `custom_list` | 自定义股票列表 | 生产环境、特定研究 | ✅ 完全支持 |
| `random_sample` | 随机抽样 | 快速验证、开发调试 | ✅ 完全支持 |
| `stratified_sample` | 分层抽样 | 因子挖掘、保证代表性 | ✅ 完全支持 |

#### 1. 全市场股票池
```yaml
stock_pool:
  type: "all_market"
```
**适用场景**：全市场因子挖掘
**跨平台兼容**：所有平台都支持

#### 2. 指数成分股（推荐用于跨平台验证）
```yaml
stock_pool:
  type: "index_components"
  index_code: "000300.XSHG"  # 沪深300
```
**适用场景**：与米筐、聚宽保持一致
**跨平台兼容**：
- ✅ 米筐：`rq.index_components("000300.XSHG")`
- ✅ 聚宽：`get_index_stocks('000300.XSHG')`
- ✅ Tushare：`ts.get_index_stocks('000300.SH')`

**支持的指数代码**：
| 指数名称 | 米筐格式 | Qlib格式 | Qlib市场 |
|---------|----------|----------|----------|
| 沪深300 | `000300.XSHG` | `SH000300` | `csi300` |
| 中证500 | `000905.XSHG` | `SH000905` | `csi500` |
| 中证1000 | `000852.XSHG` | `SH000852` | `csi1000` |
| 上证50 | `000016.XSHG` | `SH000016` | `sse50` |
| 深证成指 | `399001.XSHE` | `SZ399001` | `szse` |
| 创业板指 | `399006.XSHE` | `SZ399006` | `growth` |

**注意**：Qlib 数据需要包含对应指数的成分股数据才能使用。默认数据通常包含 `csi300` 和 `csi500`。

#### 3. 行业股票池
```yaml
stock_pool:
  type: "industry"
  industry_name: "银行"
  industry_level: 1
```
**适用场景**：特定行业研究
**跨平台兼容**：需要各平台支持行业分类
**注意**：当前版本需要数据源支持，返回空列表

#### 4. 自定义股票列表
```yaml
stock_pool:
  type: "custom_list"
  instruments:
    - "000001.XSHE"
    - "600000.XSHG"
```
**适用场景**：生产环境，固定股票池
**跨平台兼容**：所有平台都支持

#### 5. 随机抽样
```yaml
stock_pool:
  type: "random_sample"
  base_pool_type: "all_market"
  sample_size: 100
  seed: 42  # 固定种子，保证可复现
```
**适用场景**：快速验证系统
**跨平台兼容**：通过固定种子保证一致性

#### 6. 分层抽样
```yaml
stock_pool:
  type: "stratified_sample"
  base_pool_type: "all_market"
  sample_size: 500
  stratify_by: ["market_cap"]
  seed: 42
```
**适用场景**：保证样本代表性
**跨平台兼容**：需要各平台支持市值数据

## 9. 数据源切换指南（Qlib ↔ 米筐）

### 9.1 为什么需要切换数据源？

**Qlib 数据**：
- ✅ 免费开源
- ✅ 本地存储，访问速度快
- ❌ 数据截止到 2022/12/31，无法获取最新数据

**米筐 RQData**：
- ✅ 数据实时更新到最新交易日
- ✅ 支持更多数据类型（分钟线、财务数据等）
- ❌ 需要购买服务（有试用额度）
- ❌ 需要网络连接，有流量限制

### 9.2 快速切换步骤

#### 步骤1：安装米筐客户端

```bash
pip install rqdatac
```

#### 步骤2：设置环境变量

```powershell
# Windows PowerShell
setx RICEQUANT_USERNAME "你的手机号"
setx RICEQUANT_PASSWORD "你的密码"

# 验证设置
echo $env:RICEQUANT_USERNAME
```

#### 步骤3：修改配置文件

编辑 `runtime/config/env.yaml`：

```yaml
# 切换数据源
# data_source: qlib
data_source: ricequant

# 米筐配置
ricequant:
  username: ${RICEQUANT_USERNAME}
  password: ${RICEQUANT_PASSWORD}
  data_cache_dir: D:/test/factor-factory/runtime/data/ricequant_cache
```

#### 步骤4：测试连接

```bash
cd d:\test\factor-factory
python test_ricequant_connection.py
```

#### 步骤5：运行系统

```bash
cd runtime
python src/step10_iterate.py
```

### 9.3 数据源对比

| 功能 | Qlib | 米筐 RQData |
|------|------|-------------|
| **数据更新** | 截止到 2022/12/31 | 实时到最新交易日 |
| **股票列表** | `D.instruments()` | `rqdatac.all_instruments()` |
| **指数成分** | `D.instruments(market='csi300')` | `rqdatac.index_components()` |
| **行情数据** | `D.features()` | `rqdatac.get_price()` |
| **数据频率** | 日线 | 日线/分钟线 |
| **财务数据** | 有限支持 | 完整支持 |
| **费用** | 免费 | 需购买服务 |
| **流量限制** | 无 | 试用 1GB/天 |

### 9.4 常见问题

**Q: 米筐数据需要多少钱？**
A: 米筐提供试用账户（每日 1GB 流量），付费版本根据数据类型和流量配额定价，具体请访问 https://www.ricequant.com/welcome/rqdata

**Q: 可以部分使用米筐数据吗？**
A: 可以。系统支持按配置切换数据源，你可以：
- 开发和测试阶段使用 Qlib（免费）
- 生产环境使用米筐（最新数据）

**Q: 米筐和 Qlib 的数据格式有差异吗？**
A: 系统已自动处理格式转换：
- 股票代码格式统一（000001.XSHE）
- 数据索引格式统一（instrument, datetime）
- 字段命名统一（$open, $close 等）

**Q: 如何回退到 Qlib？**
A: 只需修改一行配置：
```yaml
data_source: qlib
```
无需修改任何业务代码。

### 9.5 技术实现说明

系统采用**策略模式**实现数据源切换：

```
runtime/src/data_provider/
├── base_provider.py      # 抽象基类
├── qlib_provider.py      # Qlib 实现
├── ricequant_provider.py # 米筐实现
└── provider_factory.py   # 工厂类
```

业务代码通过统一接口访问数据：
```python
from common import get_data_provider

provider = get_data_provider()
provider.initialize()

# 统一的接口，无需关心底层数据源
instruments = provider.get_instruments()
data = provider.get_price_data(...)
```
